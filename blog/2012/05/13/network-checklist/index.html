Ubuntuのネットワーク設定チェックリスト

May 13th, 2012

ネットワーク設定覚書

Ubuntuのバージョンアップ時にネットワークに繋がらなくなったことがあったため、 覚書としてメモ。

Ubuntuでネットワークに繋がらなくなった場合のチェック。

試しているVersionは

Ubuntu 11 .10 64bit

参考にさせていただいたページはこちら

http://www.math.kobe-u.ac.jp/~kodama/tips-net-setup.html

https://forums.ubuntulinux.jp/viewtopic.php?id=9522

試した手順

  • とりあえず、再起動 sudo /etc/init.d/networking restart
  • ping anyhost
  • 繋がっていないことを確認したら、/etc/network/interfaces が正しいかを確認
  • 間違っていたので修正、 その後再起動 sudo /etc/init.d/networking restart　
  • それでもつながらないので、nameserverが指定されているかを/etc/resolv.confで確認　→ 空になっている
  • /etc/resolv.conf に nameserver を追記し、sudo /etc/init.d/networking restart でつながるようになった。

今後の方針

調べてみると、根本原因はNetwork-Managerが/etc/resolv.confを書き換えることだった。

Network-Managerを削除したりするのではなく、有効活用したほうがいいのだろうと思い検索したところ、次のページを発見。参照し設定。 http://www.usupi.org/sysad/173.html

これでしばらく様子を見る。
