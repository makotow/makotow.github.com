Glossary

Feb 9th, 2013

日々に作成中。 間違っている可能性もあります。 適宜、見直し・修正を行なっています。

OS

CIFS (Common Internet File System)

MSが開発したインターネット経由でファイルを共有するプロトコル。 SMB(Server Meesage Block)をベースに開発。SBMはNetBIOSを使用しているが、CIFSはTCP/IPを使用する。 CIFSは正式にドキュメントを公開しているため、Windows以外のOSやインターネット上を介してファイル共有サービス等を使用できる。 CIFSを使用するアプリケーションとしてSambaがある。

Storage

SCSI

ハードウェア間のデータのやり取りをするインターフェース企画CPUの負荷を抑えることができる。

iSCSI

SCSIプロトコルをTCP/IPで使用する規格。

Network

NIC Teaming

物理NICを１つの仮想的なNICとして束ねる技術。 耐障害性の強化、使用帯域の向上を実現する。

Link Aggregation

LANスイッチ感を接続する複数の物理リンクを1本に束ねる技術。 トランキングと呼ぶ。 IEEE802.3ad で標準化され、CISCO用語ではEtherChannel。 NICチーミングの1つ。

TCP/IPオフロードエンジン

NICがTCP/IPの処理の一部をCPUの代わりに実行する機能。

VMWare (vSphere 5.1)

RDM (Raw Device Mapping)

仮想マシンからストレージデバイス(LUN)を直接的に扱うための仕組み。 実体は、使用するストレージデバイスのプロキシとして機能するVMFSボリュームのマッピングファイル。 VMFSファイルシステムの仮想ディスクが利用可能になるのと同時に、物理デバイスに直接アクセスできるようになる。 これによって、VMFSの管理性とRawデバイスへアクセスとを両立させている。 ほとんどの場合は、仮想ディスクのストレージにはVMFSを使用することが望ましい。

RDMを使用する場面としては、

  • 既存ファイルシステムをそのまま使用したい
  • ストレージが提供するスナップショットを使用している場合
  • MSFCを使って物理マシンをまたがるクラスタリングシステムを仮想マシンで構築する場合(共有ディスクをRDMとして構成する必要がある)

仮想互換モード

VMFSのメリットを享受できる。

物理互換モード

VMKernelが、すべてのSCSIコマンドをデバイスに渡す。 REPORT LUNのみは例外でパススルーされない。 MSFCを使用してクラスタリングシステムを構築する際には物理互換モードの必要がある。

DPM

ワークロードにあわせて起動するホストの台数を最適化すること。 DPMを設定する際に以下について考慮する - CPU,メモリを均一化したクラスタを構成 - 予期できないワークロードがある場合は、最低限必要なCPUとメモリ容量を確保する 詳細オプションMinPoweredOnCpuCapacity、MinPoweredOnMemCapacityを設定 - DRSを完全自動とし、自動化レベルを保守的にしないようにする。ホスト起動後の仮想マシンバランスを最適化するため。 - テンプレートが登録されているホストをDPMがパワーオフしてしまうことを避けるためテンプレートは1つのホストにまとめ、そのホストのDPMは無効化する。

HA と DPM

DPM、DRSはHAのアドミッションポリシーに従う。 アイドル状態のホストがあったとしても、HAの設定に違反する場合は仮想マシンの電源起動やホストの電源停止などは行われない。

また、DPMはHAが有効になっているクラスタないではアドミッションポリシーが有効になっているかどうかにかかわらず、最低限2台のホストの電源は起動した状態を維持する

DRS (Distributed Resource Scheduler)

常に最適なパフォーマンスを得られるように、vMotionを利用して自動的に仮想マシンを再配置し、 システム全体のパフォーマンスを維持する機能。 同様にストレージの負荷を平準化するために、仮想マシンが格納されているデータ領域を自動的に再配置するストレージDRSも用意されている。

EVC (Enhanced vMotion Compatibility)

EVCを利用すること異なる CPU 世代間の vMotion による移行を可能とする。 新しいCPU世代のサーバーをクラスタに追加した場合でもvMotionが可能な構成をとることができる。

通常CPU世代が異なるホスト間では、vMotionを行うことはできない。 vCenterはvMotionを行う前にホスト間でCPU命令セットに互換性があるかチェックし、互換性のないホスト間ではvMotionを実行できないようにする。

要件として、

  • ESXiはクラスタ内にあること
  • クラスタのホストCPUはAMDまたはIntelのどちらかであること
  • クラスタのすべてのホストがvMotionの要件を満たしていること
  • ホストCPUが使用するEVCモードを満たすこと
  • ハードウェ仮想化サポートとNX/XD がBIOSで有効になっていること

see also: KnowledgeBase

Storage vMotion

VAAI

vStorage API for Array Integrationの略。 共有ストレージ装置が保有するさまざまな機能を、ハイパーバイザと連携させることで有効活用するための機能

VSA Manager

VMWare vSphere Storage Applications Manager

バルーンドライバ

VMWare ESXiのメモリのオーバーコミットを実現する技術の1つ。 ゲストOSにVMWare tools をインストールした際にインストールされる。 ESXiと連携しアイドル状態の仮想マシンからメモリ税の税率にしたがってメモリ解放を行う。

VSS (Volume Shadow Copy Service)

Data Recoveryで使用する静止メカニズム。 MSが開発し、WindowsServer2003移行のOSで実装されているスナップショット技術。 VMWare ToolsにはVSSのリクエスタ機能が含まれている。

sync ドライバ

Data Recoveryで使用する静止メカニズム。

ネットワークI/Oコントロール

iSCSCIアダプタ

適用型

ソフトウェア型

ハードウェア型

ストレージIOコントロール(SIOC)

ポリシーベースでストレージI/Oの優先度を制御できる。 ディスクI/Oに対してシェア値を設定するとデータストアにたいして発行するI/Oキューをホスト間で協調して調整する。
